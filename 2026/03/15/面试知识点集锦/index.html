一、Java 基础和面向对象

面向对象：

1. 封装

  1. 封装是指把数据和操作数据的方法封装在类中，并隐藏内部实现细节
  2. 为什么要封装：
    1. 隐藏内部实现
    2. 提高安全性
    3. 提高代码可维护性

2. 继承

  1. 子类继承父类的属性和方法
  2. Java 为什么是单继承：
    避免菱形继承问题，假如同时继承两个父类中拥有同名方法，子类会不知道该调用谁

3. 多态

  1. 父类引用指向子类对象，在运行时根据对象类型决定调用的方法
  2. 多态的三个条件：
    1. 继承关系
    2. 方法重写
    3. 父类引用指向子类对象
  3. 多态的好处：
    提高代码的拓展性、降低耦合

4. 抽象

  1. 抽象是指只定义规范，不实现细节
  2. 抽象两种实现方式：抽象类、接口

5. 接口 VS 抽象类：

  1. 关键字：抽象类(abstract class)、接口(interface)
  2. 继承：抽象类(单继承)、接口(多实现)
  3. 方法：抽象类(可以有抽象方法和普通方法)、接口(默认是抽象方法，Java8 以后可以有 default 和 static 方法)
  4. 成员变量：抽象类(可以有)、接口(接口成员变量默认 public static final)

6. 重载 VS 重写

  1. 重载(Overload)：方法名相同，参数不同、与返回值无关
  2. 重写(Override)：子类重写父类方法、参数必须相同、返回值必须兼容，访问权限不能更小

7. this VS super

  1. this 表示当前对象
  2. super 表示父类对象

8. final

  1. final 修饰的基本类型不能修改，引用类型引用地址不能变，对象内容可以变
  2. final 修饰的方法不能被重写
  3. final 修饰的类不能被继承

9. equlas VS ==

  1. == 对于基本数据类型比较的是值；对于对象类型比较的是地址
  2. equlas() 是 Object 类的方法，默认比较地址，但很多类如 String 重写了 equals()方法，比较的是内容

Java 基础：

1. String 为什么不可变？

  1. String 底层的字符数组被 final 修饰，引用地址不能改变
  2. String 类被 final 修饰，不能被继承，保证行为不被修改
  3. String 没有提供修改内部字符数组的方法，因此一旦创建内容就无法被改变

2. String 不可变的好处

  1. 可以安全使用字符串常量池，提高内存利用率
  2. 天然线程安全
  3. 可以安全作为 HashMap 的 key，因为 hash 值不会变化

3. StringBuilder 为什么线程不安全

  1. StringBuilder 的底层字符数组没有被 final 修饰
  2. StringBuilder 提供了 append()等修改方法

4. 字符串常量池

String 使用字符串常量池来提高内存的利用率，当 String a = “abc” 时，JVM 会先检查字符串常量池是否存在 “abc”，如果不存在则会创建，如果存在则会直接返回引用

5. new String

a == b 的结果是 false，因为 a 指向字符串常量池的对象，而 new String(“abc”),会在堆中创建一个新的 String 对象，所有两个引用地址不相同
a.equals(b)的结果是 true，因为 String 重写 equals()方法，比较的是字符串内容，而不是地址
